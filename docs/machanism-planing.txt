🖼️ IMAGE ZOOM MECHANISM
===========================================================
The item details page features a dual-mode image zoom system that provides an optimal experience for both touch and cursor-based devices.

👆 1. MOBILE (Touch-based Zoom)
-----------------------------------------------------------
-   **Module:** `shared/utils/image-zoom.js`
-   **Activation:** Active on all devices by default for touch events.
-   **Functionality:**
    -   **Pinch to Zoom:** Users can use two fingers to zoom in and out of the image.
    -   **Pan:** When zoomed in, users can pan the image by dragging it with one finger.
    -   **Double-tap Zoom:** A quick double-tap toggles between a zoomed-in state and the default view.

🖱️ 2. DESKTOP (Cursor-based Zoom)
-----------------------------------------------------------
-   **Module:** `shared/utils/cursor-zoom.js`
-   **Activation:** Activates only on screens wider than `1024px`.
-   **Functionality:**
    -   **Hover to Zoom:** When the user hovers the cursor over the main image, a "lens" appears.
    -   **Result Box:** A separate "result box" appears next to the image, displaying the magnified portion.
    -   **Responsive Sizing:** The result box's size dynamically matches the size of the `#item-info` container, making it fully responsive.
    -   **Configurable Zoom:** The zoom level can be easily adjusted by changing the `zoomFactor` variable in `cursor-zoom.js`.

🔧 Orchestration (`item-details.js`)
-----------------------------------------------------------
-   `initializeItemDetailsPage` checks the screen width using `window.matchMedia`.
-   It dynamically imports and initializes the appropriate zoom module (`image-zoom.js` for touch, and `cursor-zoom.js` for desktop).
-   It includes cleanup functions (`cleanupImageZoom`, `cleanupDesktopZoom`) to remove event listeners when the user navigates to another item, preventing memory leaks and bugs.
